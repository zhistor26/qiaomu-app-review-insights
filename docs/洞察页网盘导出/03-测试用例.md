# 洞察页网盘导出 — 测试用例（测试金字塔）

> **关联**：[01-需求](./01-需求.md) · [02-架构设计](./02-架构设计.md) · [网盘对接-测试用例](../网盘对接-测试用例.md)（save-to-disk 基础用例）  
> 版本：v0.1 · 2026-06-17  
> 方法论：**测试金字塔** — 底层多而快，顶层少而精

---

## 0. 测试金字塔总览

```
                    ┌─────────────────────────┐
                    │  E2E / 手工验收          │  少（~8 条）
                    │  洞察页 + 更新 + 真网盘   │  慢、必跑提审
                    ├─────────────────────────┤
                    │  集成测试                │  中（~7 条）
                    │  新 API + save 封装      │  CI 可跑
                    ├─────────────────────────┤
                    │  单元测试                │  多（~14 条）
                    │  MD 生成器 / 启用条件     │  最快、最多
                    └─────────────────────────┘
```

| 层级 | 占比建议 | 运行方式 | 通过门槛 |
|------|----------|----------|----------|
| **L1 单元** | ~60% | `npm test` | P0 100% |
| **L2 集成** | ~25% | `npm test`（mock cache / fetch） | P0 100% |
| **L3 E2E** | ~15% | 微服手工 + 截图 | P0 全过方可提审 |

**原则**：MD 章节、数字一致性在 L1/L2 覆盖；inject / 真网盘只在 L3 验。

## 1. 测试范围

|----------|------------|

| 网盘 .md 与页面核心数字一致（L3） | 性能压测 |

## 3. L1 — 单元测试

> **目标**：纯函数；毫秒级；不启动 Next / Docker。

### 3.1 用例总览

| ID | 模块 | 用例 | 优先级 |
|----|------|------|--------|
| UT-INS-001 | `CachedInsightReportGenerator` | 含 `# 抖音商城 评价分析` 标题 | P0 |
| UT-INS-002 | 同上 | 含样本数 `160`、均分 `2.03` | P0 |
| UT-INS-003 | 同上 | 含 `executiveSummary` 正文 | P0 |
| UT-INS-004 | 同上 | 含 Problem / Opportunity / Action 章节标题 | P0 |
| UT-INS-005 | 同上 | 含至少 1 条评论证据（title + rating） | P0 |
| UT-INS-006 | 同上 | 含 `canonicalUrl` 或页面链接 | P1 |
| UT-INS-007 | 同上 | 含数据来源 RSS / 样本边界说明 | P1 |
| UT-INS-008 | 同上 | 含版本诊断 insights（高风险版本等） | P1 |
| UT-INS-009 | 同上 | `insights=null` 时摘要为占位文案，仍有 stats | P0 |
| UT-INS-010 | 同上 | 评论证据条数 ≤ `page.reviews.length` | P1 |
| UT-INS-011 | `canSaveInsightReport` | totalReviews=0 → false | P0 |
| UT-INS-012 | 同上 | totalReviews>0 → true（无 insights 也为 true） | P0 |
| UT-INS-013 | `buildReportFilename` | 洞察页与管理端文件名规则一致（复用 UT-001～003） | P0 |
| UT-INS-014 | 同上 | 非法 appName 字符 sanitize（复用已有用例） | P1 |

### 3.2 示例（UT-INS-002）

```typescript
import { describe, it, expect } from 'vitest';
import { CachedInsightReportGenerator } from '../cached-insight-report';
import { douyinFixture } from './fixtures/cached-app-review-page';

describe('CachedInsightReportGenerator', () => {
  it('UT-INS-002: 样本概览数字与 page 一致', () => {
    const md = CachedInsightReportGenerator.generate(douyinFixture);
    expect(md).toContain('160');
    expect(md).toContain('2.03');
    expect(md).toMatch(/73%|73 %/); // 与 formatPercent(negativeShare) 一致
  });
});
```

### 3.3 L1 通过标准

- [ ] 全部 **P0** UT-INS 绿
- [ ] `npm test` 增量 ≤ 30s

## 5. L3 — E2E / 手工验收

> **目标**：真微服 + 真洞察页 + 真网盘；提审与归档凭据。

### 5.1 环境

| 项 | 要求 |
|----|------|
| 部署 | LPK `lpk install` 成功 |
| URL | `https://appreview.<box>.heiyu.space/apps/cn/{appId}` |
| 样例 App | 抖音商城或任意有 ≥50 条样本的 CN App |
| 客户端 | 懒猫微服 WebShell 已登录 |

### 5.2 用例总览

| ID | 名称 | 优先级 |
|----|------|--------|
| E2E-INS-001 | 洞察页可见「保存至懒猫网盘」按钮 | P0 |
| E2E-INS-002 | 无评论样本页按钮 disabled | P0 |
| E2E-INS-003 | 更新洞察后页面数字刷新 | P0 |
| E2E-INS-004 | 保存到网盘主路径 ★ | P0 |
| E2E-INS-005 | 网盘 MD 与页面核心数字一致 ★ | P0 |
| E2E-INS-006 | 出现「本地 / 懒猫微服」选择器 ★ | P0 |
| E2E-INS-007 | **更新洞察 → 再导出**，MD 摘要/时间与新版一致 ★ | P0 |
| E2E-INS-008 | 用户取消保存无报错 | P1 |
| E2E-INS-009 | 管理端仪表盘网盘导出仍可用（回归） | P1 |
| E2E-INS-010 | 激励/留证截图齐全 | P1 |

### 5.3 详细用例（P0）

#### E2E-INS-001 按钮可见

| 步骤 | 预期 |
|------|------|
| 打开 `/apps/cn/{appId}` 洞察页 | 头部有「保存至懒猫网盘」，与「更新洞察」并列 |

#### E2E-INS-004 保存到网盘 ★

| 步骤 | 预期 |
|------|------|
| 点击「保存至懒猫网盘」 | ≤5s 进入选择流程 |
| 选「懒猫微服」→ 目录 → 确认 | 网盘出现 `{AppName}-分析报告-{date}.md` |

#### E2E-INS-005 MD 与页面一致 ★

| 检查项 | 预期 |
|--------|------|
| 标题 | `# {AppName} 评价分析` |
| 样本数 | 与页面「评论样本」一致 |
| 均分 | 与页面「样本均分」一致 |
| 摘要 | 与页面「摘要」段落一致（或占位说明） |
| 痛点 | 至少 1 条与页面 Problem 卡片 title 一致 |
| 评论 | 至少 1 条与「评论证据」列表一致 |

#### E2E-INS-007 更新后再导出 ★

| 步骤 | 预期 |
|------|------|
| 记录当前摘要/更新时间 | — |
| 点击「更新洞察」，等待刷新完成 | 页面 `更新于` 变化 |
| 再次「保存至懒猫网盘」 | 新 MD 的摘要/时间与**刷新后页面**一致，不是旧版 |

#### E2E-INS-010 留证截图

至少 4 张：

1. 洞察页「保存至懒猫网盘」按钮  
2. 本地 / 懒猫选择器  
3. 网盘内 `.md` 文件  
4. MD 预览（含摘要 + 1 条评论证据）
