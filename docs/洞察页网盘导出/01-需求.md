# 洞察页网盘导出 — 需求文档（PRD）

> **目标**：在公开缓存洞察页（如 `/apps/cn/{appId}`）增加「保存至懒猫网盘」，导出与当前页面一致的 Markdown 报告。  
> **原则**：复用已有网盘 inject 与 `save-to-disk` 能力，只补「页面 → MD → 网盘」缺口。  
> **关联**：[网盘对接需求](../网盘对接需求.md)（管理端仪表盘已落地）· [02-架构设计](./02-架构设计.md) · [03-测试用例](./03-测试用例.md)

---

## 1. 背景

| 项 | 说明 |
|----|------|
| 用户场景 | 在首页搜索 App → 打开「抖音商城 评价分析」等缓存洞察页 → 阅读摘要、版本诊断、痛点/机会/行动建议、评论证据 |
| 已有能力 | `analysis-dashboard.tsx`（管理端 `/reviews/[id]`）已有「保存至懒猫网盘」，走 `GET /api/apps/{id}/export-report` + `showSaveFilePicker` |
| 缺口 | **缓存洞察页**（`src/app/apps/[country]/[appId]/page.tsx`）无导出入口；现有 `export-report` API 依赖 `storage.getApps()` 的「已添加 App」，与缓存页数据源（`CachedAppReviewPage` / 文件缓存）不一致 |
| 平台要求 | 懒猫微服上架需文件选择器接入；inject 与保存逻辑已在 [网盘对接](../网盘对接需求.md) 中完成，本专题只扩展**导出入口与 MD 内容** |

### 1.1 两条产品面对比

| 维度 | 管理端分析页 | 缓存洞察页（本专题） |
|------|-------------|---------------------|
| 路由 | `/reviews/{id}` 等 | `/apps/{country}/{appId}` |
| 组件 | `analysis-dashboard.tsx` | `page.tsx`（Server Component） |
| 数据 | `AggregatedAnalysis` + 本地 storage App | `CachedAppReviewPage`（cache.ts） |
| 导出 API | `/api/apps/{id}/export-report` ✅ | **无** ❌ |
| 网盘按钮 | 已有 ✅ | **无** ❌ |
| MD 内容 | `ReportGenerator`（情感分布、commonIssues 等） | 需新建，**对齐页面展示结构** |

## 2. 需求目标

### 2.1 必须达成（P0）

1. 在缓存洞察页操作区（「App Store」「更新洞察」旁）增加按钮：**「保存至懒猫网盘」**。
2. 点击后弹出懒猫文件选择器（本地 / 懒猫微服），用户选网盘目录后写入 **一份 `.md` 文件**。
3. 导出内容与**当前页面可见信息一致**（见 §4.2），不是管理端那套 `AggregatedAnalysis` 模板。
4. 文件名规则与现有一致：`{AppName}-分析报告-{YYYY-MM-DD}.md`（复用 `buildReportFilename`）。
5. 无有效洞察数据时按钮 disabled，并提示「请先更新洞察或等待 AI 分析完成」。

### 2.2 应当达成（P1）

1. inject 不可用时回退本地下载（复用 `saveMarkdownReportToDisk` 的 `fallback-download`）。
2. MD 内含页面 URL（canonical）与更新时间，便于团队归档后对账。
3. 评论证据导出页面上展示的样本（当前最多约 80 条），不强行塞全量缓存。

### 2.3 明确不做（Out of Scope）

- ❌ 改 inject 脚本或 manifest 网盘机制（已在网盘对接专题完成）
- ❌ PDF / HTML / 图片导出
- ❌ 批量导出多个 App
- ❌ 定时任务自动写网盘
- ❌ 替换或删除管理端已有导出按钮
- ❌ 把缓存页数据写入 storage「已添加 App」列表

## 3. 用户故事

**作为** 懒猫微服上的产品/运营同学  
**我希望** 在「抖音商城 评价分析」这类洞察页一键把完整结论存进家庭网盘  
**以便** 用 Markdown 在飞书/Notion/本地归档，并与同事共享同一份带评论证据的分析  

**验收**：从打开洞察页 → 点击保存 → 网盘出现 `.md` → 打开后结构与页面一致，全程 ≤ 2 分钟。

## 4. 功能说明

### 4.1 入口与交互

**位置**：`src/app/apps/[country]/[appId]/[[...slug]]/page.tsx` 头部操作区

```
[App Store ↗]  [更新洞察]  [保存至懒猫网盘]   ← 新增
```

**交互流程**：

```
用户打开 /apps/cn/{appId}/...
  → 页面已加载 CachedAppReviewPage
  → 点击「保存至懒猫网盘」
  → GET /api/appstore/{country}/{appId}/export-report?format=markdown
  → Blob → showSaveFilePicker（inject 拦截 → 懒猫网盘）
  → 网盘出现「抖音商城-分析报告-2026-06-17.md」
```

**状态**：

| 条件 | 按钮 |
|------|------|
| `page.stats.totalReviews > 0` 且（有 `insights` 或至少有 stats + reviews） | 可点击 |
| 无评论样本 | disabled + tooltip |
| 请求中 | loading「保存中…」 |
| 用户取消选择器 | 静默，不 toast |
| API 4xx/5xx | toast「导出失败，请稍后重试」 |

### 4.2 导出 MD 应包含的章节（与页面对齐）

按用户看到的抖音商城洞察页结构组织，示例大纲：

```markdown
# {AppName} 评价分析

## 元信息
- 国家/地区、开发者、更新时间、模型、页面链接

## 样本概览
- 样本均分、评论样本数、好评/差评占比
- 星级分布（5~1 星数量）

## 数据来源
- RSS / 页面补样本 / Storefront 构成（与 ReviewSourceBreakdownPanel 一致）
- 样本边界说明文案

## 摘要
- executiveSummary（AI 执行摘要）

## 版本与口碑诊断
- 高风险版本、主导痛点、差评峰值（来自 diagnostics）
- 版本样本列表（version · 条数 · 均分）

## 洞察矩阵
### 核心痛点（Problem）
### 产品机会（Opportunity）
### 正向信号（Strength）
### 用户分层（Audience）
### 版本风险（Risk）
### 行动建议（Action）
（每条含：标题、摘要、优先级、证据引用）

## 评论证据
- 页面上展示的精选评论列表（标题、评分、版本、时间、来源、正文）

## 附录
- 生成时间、数据来源说明、免责声明（与页脚 FAQ 语气一致）
```

> **关键原则**：用户说「就是一个 ok 的 md 文档」——可读、可归档、与页面信息**不矛盾**即可；不要求复刻图表（热力图、时间线），用文字/表格表达诊断结论。

### 4.3 与现有代码的关系

| 模块 | 本专题用法 |
|------|-----------|
| `src/lib/lazycat/save-to-disk.ts` | **复用** `saveMarkdownReportToDisk`，传入自定义 `fetchReport` |
| `src/lib/lazycat/report-filename.ts` | **复用** 文件名规则 |
| `public/lazycat-injects/lzc-file-chooser-inject.js` | **不改**，已在 manifest 注入 |
| `src/lib/report/generator.ts` | **不直接复用**（面向 AggregatedAnalysis）；新建 `cached-insight-report.ts` 或扩展 generator |
| `src/lib/appstore/cache.ts` | **读** `readCachedReviewPage` / 导出 API 内复用同数据源 |
| `src/components/app-review/insight-cards.tsx` | 洞察结构参考 `ReviewInsights` 类型 |

### 4.4 建议新增 API

```
GET /api/appstore/[country]/[appId]/export-report?format=markdown
```

- 读取缓存页：`readCachedReviewPage(country, appId)`，无缓存时可按需 `generateCachedReviewPage`（与页面 SSR 一致）
- 响应：`text/markdown`，`Content-Disposition: attachment`
- 404：App 不存在或无样本

## 5. 非功能需求

| 项 | 要求 |
|----|------|
| 兼容 | 懒猫微服 WebShell 内浏览器；SSR 页面 + Client 按钮组件 |
| 体积 | 160 条样本 + 80 条证据，MD 通常 < 200KB |
| 性能 | API 生成 < 3s；保存 < 5s |
| 安全 | 报告不含 LLM API Key；仅公开评论与聚合洞察 |
| 隐私 | MD 不含微服用户身份信息 |

## 6. 验收标准

### 6.1 功能

- [ ] 打开 `/apps/cn/{某 AppId}` 洞察页，可见「保存至懒猫网盘」
- [ ] 点击后出现「本地 / 懒猫微服」选择（微服内）
- [ ] 网盘文件名为 `{AppName}-分析报告-{日期}.md`
- [ ] MD 含：摘要、样本统计、数据来源、版本诊断、Problem/Opportunity/Action 等章节、评论证据
- [ ] 与页面展示的核心数字一致（样本数、均分、差评占比等）
- [ ] 无洞察时按钮不可用或有明确提示

### 6.2 回归

- [ ] 管理端 `analysis-dashboard` 原有网盘导出仍正常
- [ ] inject 失效时仍可本地下载 fallback

### 6.3 留证（上架 / 激励）

1. 洞察页「保存至懒猫网盘」按钮截图  
2. 网盘内 `.md` 文件列表  
3. MD 内容预览（含摘要 + 至少 1 条评论证据）

## 8. 风险与对策

| 风险 | 对策 |
|------|------|
| 缓存页与管理端 MD 结构两套维护 | 共用 `ReviewInsights` / `ReviewStats` 类型；生成器单测对齐 fixtures |
| 页面 Server Component 无法直接调 picker | 独立 Client 子组件 |
| 无 AI insights 时导出太薄 | 仍允许导出 stats + reviews + diagnostics；摘要用占位说明 |
| 与旧 `export-report` 混淆 | API 路径区分 `/api/apps/` vs `/api/appstore/` |

## 9. 成功定义

**Done = 用户在缓存洞察页（如抖音商城评价分析）一键把与页面一致的 Markdown 保存进懒猫网盘，且 MD 打开即可阅读、归档。**
