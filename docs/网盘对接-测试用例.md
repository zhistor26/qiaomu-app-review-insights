## 0. 测试金字塔总览

```
                    ┌─────────────────────┐
                    │   E2E / 手工验收     │  少（~6 条）
                    │   微服 + 真网盘      │  慢、贵、必跑
                    ├─────────────────────┤
                    │   集成测试           │  中（~8 条）
                    │   API + mock 选择器  │  CI 可跑
                    ├─────────────────────┤
                    │   单元测试           │  多（~15 条）
                    │   纯函数 / 错误映射   │  最快、最多
                    └─────────────────────┘
```

| 层级 | 占比建议 | 运行方式 | 通过门槛 |
|------|----------|----------|----------|
| **L1 单元** | ~60% | `npm test` 本地 / CI | 100% 通过方可合并 |
| **L2 集成** | ~25% | `npm test`（mock 外部依赖） | 100% 通过方可合并 |
| **L3 E2E** | ~15% | 微服上手测 + 截图 | P0 全过方可提审 |

**原则**：能在 L1 测的不要放到 L3；网盘/inject 只能 L3 验，但报告生成、文件名、错误分支必须在 L1/L2 盖住。

## 1. 测试范围

| 在范围内 | 不在范围内 |
|----------|------------|
| 保存逻辑、文件名、错误码（L1） | CSV 导出网盘 |
| export-report API 契约（L2） | file_handler 打开 |
| mock `showSaveFilePicker` 流程（L2） | 定时写网盘 |
| 微服真网盘保存（L3） | 多浏览器矩阵 |
| inject 合规弹窗（L3） | 性能压测 |

## 2. 测试基础设施（实现阶段添加）

| 项 | 选型 |
|----|------|
| 运行器 | Vitest（推荐）或 Node built-in test |


## 3. L1 — 单元测试

> **目标**：纯函数、无 IO 或 IO 全 mock；毫秒级；数量最多。

### 3.1 用例总览

| ID | 模块 | 用例 | 优先级 |
|----|------|------|--------|
| UT-001 | `report-filename.ts` | 生成 `{appName}-分析报告-{date}.md` | P0 |
| UT-002 | `report-filename.ts` | appName 含 `/\:*?"<>|` 时 sanitize | P0 |
| UT-003 | `report-filename.ts` | appName 为空时用 fallback | P1 |
| UT-004 | `save-to-disk.ts` | `mapExportError(404)` → EXPORT_FAILED | P0 |
| UT-005 | `save-to-disk.ts` | `mapExportError(500)` → EXPORT_FAILED | P0 |
| UT-006 | `save-to-disk.ts` | `AbortError` → USER_CANCELLED | P0 |
| UT-007 | `save-to-disk.ts` | 无 `showSaveFilePicker` → 走 fallback-download | P0 |
| UT-008 | `canSaveReport(analysis)` | totalReviews=0 → false | P0 |
| UT-009 | `canSaveReport(analysis)` | totalReviews>0 → true | P0 |
| UT-010 | `ReportGenerator` | markdown 含 `# {name} 用户评论分析报告` | P0 |
| UT-011 | `ReportGenerator` | markdown 含 totalReviews 数字 | P0 |
| UT-012 | `ReportGenerator` | format=html / summary 非空 | P1 |

### 3.2 示例（UT-001）

```typescript
// src/lib/lazycat/__tests__/report-filename.test.ts
import { describe, it, expect } from 'vitest';
import { buildReportFilename } from '../report-filename';

describe('buildReportFilename', () => {
  it('UT-001: 标准文件名', () => {
    expect(buildReportFilename('ChatGPT', new Date('2026-06-16')))
      .toBe('ChatGPT-分析报告-2026-06-16.md');
  });

  it('UT-002: 非法字符替换', () => {
    expect(buildReportFilename('App/Name:Test', new Date('2026-06-16')))
      .toBe('App-Name-Test-分析报告-2026-06-16.md');
  });
});
```

### 3.3 L1 通过标准

- [ ] 全部 **P0** UT 绿
- [ ] `npm test` ≤ 30s

## 4. L2 — 集成测试

> **目标**：跨模块、mock 平台 API；不依赖真微服/真网盘。

### 4.1 用例总览

| ID | 范围 | 用例 | 优先级 |
|----|------|------|--------|
| IT-001 | export-report API | 存在 app + 有分析 → 200 + `text/markdown` | P0 |
| IT-002 | export-report API | app 不存在 → 404 + JSON error | P0 |
| IT-003 | export-report API | `format=html` → `text/html` | P1 |
| IT-004 | export-report API | `format=summary` → `text/plain` | P1 |
| IT-005 | save-to-disk | mock fetch 200 → 调用 showSaveFilePicker | P0 |
| IT-006 | save-to-disk | mock fetch 404 → 抛 EXPORT_FAILED，不调 picker | P0 |
| IT-007 | save-to-disk | picker 抛 AbortError → USER_CANCELLED | P0 |
| IT-008 | save-to-disk | writable.write 收到与 fetch body 相同字节 | P0 |

### 4.2 示例（IT-005）

```typescript
// src/lib/lazycat/__tests__/save-to-disk.test.ts
it('IT-005: 成功拉报告并打开保存选择器', async () => {
  const markdown = '# Test 用户评论分析报告\n';
  global.fetch = vi.fn().mockResolvedValue({
    ok: true,
    blob: () => Promise.resolve(new Blob([markdown], { type: 'text/markdown' })),
  });
  const write = vi.fn().mockResolvedValue(undefined);
  const close = vi.fn().mockResolvedValue(undefined);
  global.showSaveFilePicker = vi.fn().mockResolvedValue({
    createWritable: () => Promise.resolve({ write, close }),
  });

  const result = await saveMarkdownReportToDisk('app-1', 'Test');

  expect(result.mode).toBe('file-picker');
  expect(showSaveFilePicker).toHaveBeenCalledOnce();
  expect(write).toHaveBeenCalled();
});
```

### 4.3 L2 通过标准

- [ ] 全部 **P0** IT 绿
- [ ] 不启动 Docker / 不连微服

## 5. L3 — E2E / 手工验收

> **目标**：真微服 + 真 inject + 真网盘；**数量少**；提审与激励凭据。

### 5.1 环境

| 项 | 要求 |
|----|------|
| 微服 | LPK 已 install，`app status` = Installed |
| 客户端 | 懒猫微服已登录；开发者工具可用 |
| URL | `https://appreview.<box>.heiyu.space` |
| 数据 | 至少 1 个 App 已完成分析 |

### 5.3 详细用例（P0 摘要）

## 7. 执行顺序（推荐）

```
开发中  →  UT (L1) 随写随跑
PR 前   →  UT + IT 全绿 (npm test)
合并后  →  build + install
提审前  →  E2E P0 全过 + 截图 (E2E-011)
```
