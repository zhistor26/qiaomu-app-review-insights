# 引流剔除需求文档（PRD）

> **目标**：Fork 后移除上游「向阳乔木 / Qiaomu 个人生态」全部引流，使应用在懒猫微服自托管场景下无外链导流、无打赏/公众号、无官方域名 SEO 绑定。  
> **原则**：功能零损失（App 搜索、洞察生成、缓存页、榜单、限流逻辑保留），仅删改展示层与元数据。  
> **关联**：[架构设计](./引流剔除-架构设计.md) · [测试用例](./引流剔除-测试用例.md)

---

## 1. 背景

| 项 | 说明 |
|----|------|
| 上游项目 | [qiaomu-app-review-insights](https://github.com/joeseesun/qiaomu-app-review-insights)（MIT） |
| 问题 | 页脚外链网、打赏/公众号弹窗、限流弹打赏、SEO/统计绑定 `*.qiaomu.ai`、README/上架元数据推广作者 |
| 使用场景 | 懒猫微服 LPK 自托管，用户不应被导向作者 GitHub、个人站、推荐站、X、微信 |
| 本文范围 | **只做一件事**：清点并剔除全部引流触点 |

---

## 2. 引流定义

| 类型 | 定义 | 处理方式 |
|------|------|----------|
| **A. 强引流** | 主动导向作者个人站、社交、公众号、打赏、配套 Skill | **必须删除** |
| **B. 域名/SEO 绑定** | 硬编码 `appreview.qiaomu.ai`、`qiaomu.ai` 于 canonical/sitemap/JSON-LD | **改为 env 或 localhost** |
| **C. 统计回传** | Umami 脚本指向 `umami.qiaomu.ai` | **删除或 env 化** |
| **D. 文档/上架推广** | README、package.yml 中的 Demo、作者节、GitHub homepage | **删改** |
| **E. 品牌命名** | `QIAOMU_LLM_*`、`?whois=qiaomu`、产品名「乔木」 | **本期可选**，见 §6 |

---

## 3. 需求目标

### 3.1 必须达成（P0）

1. 页面 UI 无外链至：`tuijian.qiaomu.ai`、`qiaomu.ai`、`blog.qiaomu.ai`、`github.com/joeseesun`、`x.com/vista8`。
2. 无打赏按钮、无公众号二维码弹窗、无 `/qiaomu/reward.png`、`/qiaomu/wechat.jpg` 资源引用。
3. API 429 / 生成排队时**仅展示错误/提示文案**，不弹打赏 Dialog。
4. 页脚无「Qiaomu Network」外链组、无「向阳乔木」宣传文案。

### 3.2 应当达成（P1）

1. 未配置 `NEXT_PUBLIC_SITE_URL` 时，canonical/sitemap/缓存页 URL 不出现 `*.qiaomu.ai`。
2. 页面源码无 `umami.qiaomu.ai`（除非用户自行配置统计）。
3. JSON-LD 无 `author.url: https://qiaomu.ai`。
4. README 删除 Demo badge、作者节、`npx skills add joeseesun/...`。
5. `package.yml` / `lzc-manifest.yml` 删除 `author: joeseesun`、`homepage` 指向上游 GitHub、description 中「上游链接」。

### 3.3 明确不做（Out of Scope）

- ❌ 产品全面 rebranding（改应用名、换 icon、改 package id）—— 另开专题
- ❌ 重写 `QIAOMU_LLM_*` 环境变量名 —— 可选阶段
- ❌ 替换 `?whois=qiaomu` 为正式鉴权 —— 可选阶段
- ❌ 修改 MIT LICENSE 版权行（法律信息，非 UI 引流）
- ❌ 删除 App Store 官方链接（`trackViewUrl`）—— 业务必需

## 7. 风险与依赖

| 风险 | 缓解 |
|------|------|
| 删页脚后品牌识别度下降 | 保留 `BrandMark` + 中性产品名 |
| 改 DEFAULT_SITE_URL 影响已有缓存页 URL | 部署时必设 `NEXT_PUBLIC_SITE_URL` |
| package.yml 改 author 影响商店展示 | 改为 fork 方信息 |
| 限流弹窗删除后用户不知如何解决 | 429 文案保留「今日次数已用完」 |
